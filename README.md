Design Patterns Projects
This repository contains multiple projects developed as part of a Design Patterns course. Each project focuses on applying software design patterns to real-world inspired systems, improving code structure, scalability, and maintainability.
Projects

Spore Game Simulation
A simulation inspired by evolutionary game mechanics where entities evolve, interact, and adapt over time.

**Key Features:**
- Entity creation and evolution logic  
- Interaction between organisms  
- Dynamic state changes  

**Design Patterns Used:**
- Factory (object creation)
- Strategy (behavior selection)
- Observer (state updates)

Habit Tracker API
A backend API for tracking daily habits, progress, and user activity.

**Key Features:**
- Create and manage habits  
- Track daily progress  
- Persistent storage of user data  
- REST-like API structure  

**Design Patterns Used:**
- DAO (data access abstraction)
- Builder (object construction)

Weather Monitoring System
A system that simulates weather data collection and notifies dependent components about changes.

**Key Features:**
- Real-time weather updates  
- Notification system for subscribers  
- Data processing and monitoring  

**Design Patterns Used:**
- Observer (event-driven updates)
- Decorator (extending functionality)

Technologies
- Programming Languages: Python
- Object-Oriented Programming (OOP)
- Design Patterns
  
Purpose
The goal of these projects was to gain hands-on experience with **design patterns** by applying them to practical systems such as simulations, APIs, and monitoring tools.
Skills Demonstrated
- Object-Oriented Design
- Software Architecture
- Design Patterns Implementation
- Backend Development Concepts
- System Modeling
These projects were developed as part of academic coursework and demonstrate understanding of core software engineering principles.
